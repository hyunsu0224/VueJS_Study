Title : First
< 폼 입력 바인딩의 기본 사용법 >
v-model 디렉티브를 사용하여 input, textarea, select 요소들에 양방향 데이터 바인딩을 생성할 수 있다.
v-model 디렉티브는 input type 요소를 변경하는 올바른 방법을 자동으로 선택한다.
약간 이상하지만, v-model은 기본적으로 사용자 입력 이벤트에 대한 데이터를 업데이트하는
syntax sugar이며 일부 경우에는 주의가 필요하다.
v-model은 모든 form 요소의 초기 value 와 checked 그리고 selected속성을 무시한다.
항상 Vue 인스턴스 데이터를 원본 소스로 취급한다.
component의 data 옵션 안에 있는 JS에서 초기값을 선언해야한다.
v-model은 내부적으로 서로 다른 속성을 사용하고 서로 다른 입력 요소에 대해 서로 다른 이벤트를 전송한다.
  • text와 textarea 태그는 value속성과 input 이벤트를 사용한다.
  • check Box와 radio Button은 checked속성과 change 이벤트를 사용한다.
  • select 태그는 value는 prop으로, change를 이벤트로 사용한다.

Title : Second
< 값 바인딩하기 >
라디오, 체크박스 및 셀렉트 옵션의 경우, v-modle 바인딩 값은 보통 정적인 문자열(또는 체크 박스의 boolean)
picked는 선택시 문자열 "a"
toggle는 true 또는 false 이다
selected는 ABC 선택시 abc가 나온다.
그러나 때로는 현재 활성 인스턴스의 동적 속성에 값을 바인딩하려고 할 때, v-bind를 사용할 수 있다.
또한, v-bind를 사용하면 입력 값을 문자열이 아닌 값에 바인딩 할 수 있다.

Title : Third
< 수식어 >
.lazy
기본적으로, v-model은 각 input 이벤트 후 입력과 데이터를 동기화한다.
lazy 수식어를 추가하여 change 이벤트 이후에 동기화를 할 수 있다.
input 대신 change 이후에 동기화됩니다.

.number
사용자 입력이 자동으로 숫자로 형변환 되기를 원한다면 v-model이 관리하는 input에 number수식어를 추가하자.

.trim
사용자가 입력한 내용에서 자동으로 앞뒤 공백을 제거하는 트림처리가 되길 바란다면, v-model이 관리하는 input에 trim을 추가
Title : Fourth
< 컴포넌트에서의 v-model >
HTML의 기본 제공 input type은 항상 사용자의 요구를 만족시킬 수는 없다.
Vue 컴포넌트는 완전히 사용자 정의된 동작의 재사용 가능한 input을 만들수 있고,
이 input은 v-model에서도 작동한다.
